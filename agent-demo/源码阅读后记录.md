### Commander.js 在项目中的角色

**选型**：`@commander-js/extra-typings`（Commander.js 的 TypeScript 增强版，自动推导 options 类型）

**核心作用**：CLI 框架，负责参数解析、类型校验、help 生成、命令路由。类比 Web 框架中的 Express/Koa。

**三个关键 API**：
| API | 作用 | 类比 |
|-----|------|------|
| `.option('--model <m>')` | 声明一个选项 | Express 的路由参数 |
| `.command('mcp')` | 注册子命令 | Express 的 `router.use('/mcp', ...)` |
| `.action(fn)` | 绑定处理函数 | Express 的 route handler |

**链式调用**：每个方法都 `return this`，所以可以 `.option().option().action()` 无限接龙。本质等价于多次分开调用，只是写法紧凑。

**命令树结构**（main.tsx 中的完整声明）：
```
program (claude)                    ← new CommanderCommand()
├── .name('claude')                 ← 显示名（用于 --help）
├── .option('--model/--print/...')  ← 50+ 选项声明（1071-1119 行）
├── .action(...)                    ← 主处理器（1120-~4000 行，2800 行核心逻辑）
│
├── program.command('mcp')          ← 子命令（4006 行）
│   ├── mcp.command('serve/list/add-json/remove/...')
├── program.command('auth')         ← 子命令（4212 行）
│   ├── auth.command('login/logout/status')
├── program.command('plugin')       ← 子命令（4260 行）
│   ├── pluginCmd.command('validate/list/install/...')
│   └── marketplace.command('add/list/remove/update')
├── program.command('doctor/update/agents/...')
│
└── program.parseAsync(process.argv) ← 最后一步：解析 → 路由 → 执行
```

**preAction 钩子**：Commander.js 提供的全局前置钩子，所有命令执行前自动触发。
- 作用：统一初始化（init + 迁移 + 远程配置 + 插件加载）
- 类比：Express 的 `app.use(middleware)`，写一次覆盖所有路由
- Commander.js 共 4 个钩子：preAction / postAction / preSubcommand / postSubcommand，Claude Code 只用了 preAction

**子命令 vs 选项不冲突**：
- `claude mcp list` → "mcp" 是子命令（不带 `-`），匹配 `program.command('mcp')`
- `claude --model opus` → "--model" 是选项（带 `--`），匹配 `program.option('--model')`
- 解析顺序：先看有没有子命令匹配 → 没有就当根命令 + 选项处理

**执行流程**：
```
program.parseAsync(process.argv)
  → 解析参数（区分子命令 / 选项 / 位置参数）
  → 触发 preAction 钩子（全局初始化）
  → 路由到匹配的 .action() 并执行
```

**结论**：Commander.js 是基础设施，不需要深读其源码。只需理解它是"路由器 + 中间件"，真正的业务逻辑在每个 `.action()` 回调里。
