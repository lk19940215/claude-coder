# 目录重构方案

## Context

当前 `src/` 目录有 13 个文件平铺在一起，缺乏组织。需要按职责拆分为子目录，提升代码可维护性和可读性。

## 当前结构问题

```
src/
├── auth.js          # CLI 命令
├── config.js        # 核心配置
├── hooks.js         # 会话钩子
├── indicator.js     # UI 组件
├── init.js          # CLI 命令
├── plan.js          # 独立模块 + CLI
├── prompts.js       # 模板引擎
├── runner.js        # 主控
├── scanner.js       # 扫描逻辑
├── session.js       # 核心会话
├── setup.js         # CLI 命令
├── tasks.js         # 任务管理
└── validator.js     # CLI 命令
```

**问题**:
1. CLI 命令实现与核心模块混在一起
2. 无清晰的职责边界
3. 依赖关系不明确

## import 路径更新

所有文件需要更新 import 路径：

**cli.js**:
```javascript

const setup = require('../src/setup');
const { init } = require('../src/init');
const { auth } = require('../src/auth');
const validator = require('../src/validator');
const tasks = require('../src/tasks');
const { runSimplifySession } = require('../src/session');
const plan = require('../src/plan');

// After
const runner = require('../src/core/runner');
const setup = require('../src/commands/setup');
const { init } = require('../src/commands/init');

const validator = require('../src/commands/validator');
const tasks = require('../src/modules/tasks');
const { runSimplifySession } = require('../src/core/session');
const plan = require('../src/modules/plan');
```

## config.js 路径更新

```javascript
// Before
function getPromptPath(name) {
  return path.join(__dirname, '..', 'prompts', name);
}

// After
function getPromptPath(name) {
  return path.join(__dirname, '..', 'templates', name);
}
```

## 实施步骤

1. **创建目录结构**
   ```bash
   mkdir -p src/common src/core src/modules src/commands templates
   ```

2. **移动文件**（按依赖顺序，先移动无依赖的）
   - common/: config.js, indicator.js
   - core/: hooks.js, session.js, runner.js
   - modules/: tasks.js, prompts.js, scanner.js, plan.js
   - commands/: setup.js, init.js, auth.js, validator.js
   - templates/: 重命名 prompts/ 下的文件

3. **更新 import 路径**
   - 逐文件更新相对路径

4. **更新 config.js 中的 paths()**
   - 模板文件路径指向 templates/

5. **更新 cli.js**
   - 所有 require 路径更新

6. **测试验证**
   ```bash
   claude-coder --help
   claude-coder status
   claude-coder plan "test"
   ```
